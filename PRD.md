# 产品需求文档 (PRD) - 《心动坐标》(HeartSync)

**文档版本：** v1.0
**技术栈：** Starlette + SQLite3 + WebSocket + pyvenv, 在项目目录下建立.venv和requirements.txt
**核心理念：** 仪式感、私密性、游戏化互动

---

## 1. 准入与安全机制 (Access Control)
由于情侣网站的特殊性，必须建立严格的准入壁垒。

### 1.1 真爱口令 (The Passphrase)
* **功能描述：** 网站不设常规注册页面。首次部署时由开发者在数据库设置两个账号。
* **登录逻辑：**
    * 用户访问根路径 `/` 时，中间件检查 Session。
    * 若未登录，强制重定向至 `/gate`（传送门页面）。
    * **口令挑战：** 页面仅有一个输入框，需输入正确的“真爱口令”（如：初见日期或约定暗号）。
    * **错误反馈：** 输入错误时，背景颜色微红并震动，提示“只有对的人才能进入”。

### 1.2 身份隔离
* **限制：** 全局仅支持 `User_A` 和 `User_B` 两个 ID。
* **Session 持久化：** 登录后有效期设为 30 天，避免频繁输入口令。

## 2. 核心功能模块

### 2.1 恋爱计时引擎 (Love Counter)
* **数据源：** 数据库 `config` 表中的 `anniversary_date`。
* **显示逻辑：**
    * **天数：** $CurrentDate - AnniversaryDate$（取整）。
    * **分秒：** 首页支持实时跳动的秒表，显示“相爱已：X年 X月 X天 X小时 X分 X秒”。
* **纪念日提醒：** 距离 100 天、一周年、五百天等整数日子前 7 天，首页标题栏滚动提醒。

### 2.2 阶梯式成就系统 (Milestone System)
成就不是一次性的，而是随着时间自然“生长”的。

* **成就表设计：**
    | 等级 | 成就名称 | 解锁条件 | 视觉表现 |
    | :--- | :--- | :--- | :--- |
    | LV1 | 萌芽 | 相识 1 天 | 灰色种子变绿 |
    | LV2 | 默契初现 | 连续双人打卡 7 天 | 获得“小萌芽”图标 |
    | LV3 | 百日维新 | 相识 100 天 | 背景解锁“星空”主题 |
    | LV4 | 半载同行 | 相识 182 天 | 首页解锁“时光相册”功能 |
    | LV5 | 岁月如歌 | 相识 365 天 | 解锁“双人联机游戏”模块 |

* **打卡逻辑：**
    * **每日一签：** 两人每天各点击一次。
    * **共同打卡：** 当天两人都点击后，进度条推进，并触发一次爱心特效。

### 2.3 双人联机互动 (Interactive Mini-Games)
利用 Starlette 的异步特性实现实时互动。

* **模块 A：同步画板 (Canvas Sync)**
    * **场景：** 一个人画，另一个人实时看（或一起涂鸦）。
    * **技术：** WebSocket 传输坐标数据。
* **模块 B：默契问答 (Quiz)**
    * **场景：** 系统出题（如：TA 换发型了你会发现吗？），两人选 A/B。
    * **结果：** 只有两人选一致时，才增加“默契值”积分。

## 3. 补充功能设计

### 3.1 许愿池 (Bucket List)
* 记录“想一起做的 100 件事”。
* 支持上传完成后的照片，点击勾选后，该条目从“待办”进入“已达成”并归档到时光相册。

### 3.2 情绪天气 (Mood Sync)
* 用户可以设置当前情绪（开心、委屈、想抱抱）。
* 对方打开网页时，首页背景色会随对方的心情改变（如委屈时背景变为淡蓝色）。

## 4. 非功能性需求

### 4.1 UI/UX 规范
* **设计风格：** 极简新拟态 (Neumorphism) 或 毛玻璃 (Glassmorphism)。
* **响应式：** 必须适配移动端（大部分打卡行为发生在手机上）。
* **动效：** 所有的成就达成必须有 Lottie 动画或 CSS 粒子特效。

### 4.2 数据存储方案 (SQLite3 Schema)
```sql
-- 用户表
CREATE TABLE users (id INT PRIMARY KEY, name TEXT, secret_key TEXT);
-- 成就表
CREATE TABLE achievements (id INT, user_id INT, ach_name TEXT, unlock_date DATE);
-- 打卡日志
CREATE TABLE daily_checkin (id INT, user_id INT, checkin_time TIMESTAMP);
-- 全局配置（如纪念日）
CREATE TABLE meta_config (key TEXT, value TEXT);
```

### 4.3 隐私保护
* **禁止抓取：** 根目录配置 `robots.txt` 拒绝所有爬虫。
* **错误处理：** 任何未授权的 URL 访问统一重定向至 404 页面，且不暴露任何路由信息。
